Workshop 1 - Single vs Pair vs Mob

Overview

All developers work differently, all teams work differently and we all collaborate differently. Most of us know of Pair and Mob programming but do not often get the chance to compare and contrast these approaches.

During this workshop, you will have the opportunity to try collaborative coding in a number of different group sizes. We will work on a series of problems using a single coder, a pair and mob programming. Depending on attendance, we may also try out a triple and a quartet.

The Aim

The aim of the workshop is to provide attendees the opportunity to try out different collaboration approaches and to discuss:

  • how we collaborate when writing code
  • the pros and cons of different collaboration approaches

Preparation

Due to the nature of the workshop, not everyone needs to bring a laptop with an IDE, but we don't want to be short of singles and pairs so please plan to bring one if you are attending.

Workshop 2 - Clean Code

Overview

In this workshop you’ll be given a simple project with regressions tests and be asked to refactor the code to make it more maintainable and to add some new functionality.

We’ll code in pairs and then each pair will share their solution with the group.

The Aim

The aim of the workshop is to discuss:

  • how we write maintainable code others will understand
  • what factors are important in writing clean code
  • how should we structure our code to make it easy to understand
  • the pros and cons of different solutions
  • how do we balance the time spent adding new features with time spent refactoring

Preparation

If you wish to attend please bring a laptop and download the source from here.

The code has been translated into several languages so you can do the workshop in a language you’re familiar with. Please pick one version and get the code and tests running before you start the workshop. All the versions are functionally identical.
